# Правила для магазина обуви

## Структура проекта

- `myshop` отвечает за товары и категории.
- `cart` отвечает за корзину пользователей.
- `Shop` содержит глобальные настройки и маршруты.

## Маршруты

- `''` — главная страница каталога.
- `category/<slug:slug>/` — товары категории.
- `<slug:slug>/` — страница товара.
- `cart/` — страница корзины.
- `cart/add/<int:product_id>/` — добавление товара в корзину.
- `cart/remove/<int:product_id>/` — удаление товара из корзины.

## Модели

- Все slug должны быть уникальными.
- `Product.is_available` определяет видимость товара.
- `Product.image` должен отображаться через `MEDIA_URL`.
- В модели `Product` метод `get_absolute_url` должен возвращать `products:detail`.

## Шаблоны

- Каждый шаблон должен наследовать `base.html`.
- Использовать `{% load static %}` и `{% static '...' %}`.
- Использовать `{% url %}` для ссылок на маршруты.
- В шаблонах обращаться к полям модели `product.name`, `product.description`, `product.price`, `product.image.url`.

## Корзина

- Хранить корзину в сессии через `settings.CART_SESSION_ID`.
- После изменений помечать сессию как изменённую.
- Правильно считать сумму через `Decimal`.
- Не хранить объекты модели в сессии, только ID, количество и цену.
